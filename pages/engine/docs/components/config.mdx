## Components Config & `compgen` Tool

The `compgen` CLI tool generates libraries for Components. It makes it much less error-prone than using the Components low-level API, and comes with typed Move APIs when setting and retrieving records.

### Using `compgen` with the Obelisk framework

If you are using the Obelisk framework and have a `obelisk.config.ts` file in your project, you can edit your Components config directly in this file!

A Components config should be named `obelisk.config.ts`, and placed at the root of your project.

This is the minimal config:

```tsx
import { ObeliskConfig } from "@0xobelisk/common/src/codegen/types";

export const obeliskConfig = {
  name: "example", // name of the Move project
  description: "example desc", // description of the Move project
  components: {}, // an empty config with no components，
  singletonComponents: {}, // an empty config with no singleton components
} as ObeliskConfig;
```

### Generating the components

To generate the components, run `pnpm obelisk compgen ./obelisk.config.ts` in the same folder as the config file.

### Component Type

* **components**: Requires developer-defined entity_key. Obelisk will provide some common methods for generating keys.
* **singletonComponents**: The getter and setter functions will not have a `key` argument.


Currently, the component supports four types, and will be added later according to the demand.

#### component with a single column

```tsx
export const obeliskConfig = {
  name: "example",
  description: "example desc",
  components: {
      single_column1: { name: "vector<u8>" },
      single_column2: "vector<u8>"
  },
  singletonComponents: {}, // an empty config with no singleton components
} as ObeliskConfig;
```

#### component with a multi column

```tsx
export const obeliskConfig = {
  name: "example",
  description: "example desc",
  components: {
      single_column1: { name: "vector<u8>" },
      single_column2: "vector<u8>",
      multi_column: {
        name: "vector<u8>",
        age: "u8",
        sex: "bool"
    }
  },
  singletonComponents: {}, // an empty config with no singleton components
} as ObeliskConfig;
```

#### component with a single value

```tsx
export const obeliskConfig = {
  name: "example",
  description: "example desc",
  components: {
      single_column1: { name: "vector<u8>" },
      single_column2: "vector<u8>",
      multi_column: {
        name: "vector<u8>",
        age: "u8",
        sex: "bool"
    }
  },
  singletonComponents: {
      single_value:{
          type: "u64",
          init: 10 // initialise the value of my_component4
      }
  }
} as ObeliskConfig;
```

#### component with a single struct

```tsx
export const obeliskConfig = {
  name: "example",
  description: "example desc",
  components: {
      single_column1: { name: "vector<u8>" },
      single_column2: "vector<u8>",
      multi_column: {
        name: "vector<u8>",
        age: "u8",
        sex: "bool"
    },
  singletonComponents: {
    single_value: {
          type: "u64",
          init: 10
      },
    single_struct: {
          type: {
              admin: "address",
              fee: "u64",
          },
          init: {
              admin: "@0x1",
              fee: "100",
         }, // initialise the value of config
      }
  }
} as ObeliskConfig;
```
